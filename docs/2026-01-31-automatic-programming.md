# Automatic programming

## 要約

Redisの作者antirezが、AI支援によるソフトウェア開発を「Automatic Programming（自動プログラミング）」と「Vibe Coding」に区別して論じている。

### Automatic Programming（自動プログラミング）
- プログラマーが**積極的に関与**し、設計・ビジョン・方向性を持ちながらAIを活用する開発手法
- 人間が何をどう作るか理解し、AIに継続的に指示・修正を加える
- この方法で生成されたコードは**プログラマー自身のコード**である
- 高品質で、作り手のビジョンを厳密に反映したソフトウェアを目指す

### Vibe Coding
- 非常に抽象的な指示だけでAIに任せ、プログラマーが過程にほとんど関与しない開発手法
- 何が起きているか深く理解せずにソフトウェアを生成する
- これはこれで、ソフトウェア開発を民主化する価値がある

### 核心的な主張

> **「プログラミングは自動化されつつあるが、ビジョンはまだ自動化されていない」**

antirezは、Redisを例に挙げる。Redisには技術的な新規性はほとんどなく、基本的なデータ構造とネットワーキングコードの組み合わせだった。それでも有用なソフトウェアになったのは、**そこに込められたアイデアとビジョン**があったからだ。

AIが生成したコードを「自分のもの」と主張することに躊躇する必要はない。LLMの事前学習データは人間が作ったものであり、それは集合知の贈り物のようなもの。今や私たちは、ある意味で集合的な心(collective mind)で繋がっている。

## 所感

この短い文章は、AI時代のプログラマーのアイデンティティに関する重要な問いを提起している。

「AIが書いたコード」という表現は、実は**プログラマーの能動性を隠してしまう**。Automatic Programmingとして捉え直すことで、プログラマーの役割——ビジョン、設計、判断、品質管理——が明確になる。

特に「Programming is now automatic, vision is not (yet)」という一文は秀逸だ。道具が進化しても、**何を作るか、なぜ作るか、どうあるべきか**という根本的な問いは、依然として人間の領域に残る。

antirezのように長年第一線で活躍してきたプログラマーが、AI支援開発を「自分のコード」と堂々と主張する姿勢は、業界全体にとって重要な意味を持つ。技術の進化を恐れず、むしろ活用して、より高い次元での創造性を発揮する——これが次の時代のプログラマー像なのかもしれない。
